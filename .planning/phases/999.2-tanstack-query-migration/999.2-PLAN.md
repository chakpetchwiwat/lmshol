# Phase 999.2: TanStack Query Migration - PLAN

## Goal

Ship a safe, wave-based migration of frontend server-state fetching to TanStack Query v5, starting with shared infra and the lesson flow, then expanding to the rest of the user and admin surfaces.

## Exit Criteria

- TanStack Query is wired in `elearning-webapp`
- query keys exist in a shared utility
- GET wrappers needed for migrated pages accept request config for `signal`
- `LessonPlayer.jsx` is migrated without regressions in quiz flow or lesson transition UX
- Priority A and B user pages are migrated
- Core admin pages are migrated
- Build passes after each execution wave

## Execution Strategy

### Sequence
1. Build shared infrastructure
2. Migrate `LessonPlayer.jsx`
3. Verify quiz and lesson transition guardrails
4. Migrate remaining Priority A user pages
5. Migrate Priority B user pages
6. Migrate admin pages
7. Finish with reports and cleanup

### Delivery Model
- Complete one wave at a time
- Run build verification after every wave
- Do not start the next wave until the current one is stable

## Work Breakdown

### Task Group 1: Shared Infra

#### Objective
Create the foundation that every migrated page will use.

#### Tasks
- Add `@tanstack/react-query`
- Add `@tanstack/react-query-devtools`
- Create `src/lib/queryClient.js`
- Add `QueryClientProvider` to `src/main.jsx`
- Mount devtools in dev mode only
- Create `src/utils/queryKeys.js`
- Update query-facing GET wrappers in `src/utils/api.js` to accept optional config
- Confirm query functions can pass `signal` into axios requests

#### Deliverables
- `src/lib/queryClient.js`
- `src/utils/queryKeys.js`
- updated `src/main.jsx`
- updated `src/utils/api.js`

#### Verification
- app starts
- routes render
- `npm run build` passes

### Task Group 2: Lesson Flow Migration

#### Objective
Replace ad hoc fetching in `LessonPlayer.jsx` with query/mutation hooks while preserving all existing behavior.

#### Tasks
- Create `src/hooks/queries/useCourseDetail.js`
- Create `src/hooks/queries/useLessonQuestions.js`
- Create `src/hooks/queries/useLessonPlayerData.js`
- Create `src/hooks/mutations/useSubmitQuiz.js`
- Create `src/hooks/mutations/useUpdateLessonProgress.js`
- Refactor `LessonPlayer.jsx` to consume the new hooks
- Keep quiz answers and view state in local component state
- Keep document access as on-demand logic
- Add precise invalidation for:
  - course detail
  - user course list when necessary

#### Guardrails
- Keep result container after quiz submit
- Keep review flow from clearing quiz result
- Keep soft overlay loading between lessons
- Keep full-screen loader only for first load

#### Verification
- open lesson
- switch lessons quickly
- submit quiz
- review quiz
- complete lesson
- navigate to next lesson
- `npm run build`

## Dependency Map

### Hard Dependencies
- Shared infra must be done before any page migration
- Lesson flow must stabilize before other user pages inherit its pattern
- Admin CRUD pages should wait until shared invalidation conventions are established

### Soft Dependencies
- Home data hook can inform patterns for dashboard composite hooks
- Course detail invalidation strategy can inform announcement/admin detail patterns

## Risk Register

### R-01: Quiz regression
- Mitigation: migrate `LessonPlayer.jsx` in isolation and verify result-card behavior explicitly

### R-02: Lesson transition flash
- Mitigation: preserve current local loading overlay UX while changing only request orchestration

### R-03: Over-invalidation
- Mitigation: define narrow invalidation rules before migrating admin CRUD-heavy pages

### R-04: Toast duplication
- Mitigation: keep query error UI separate from mutation toast handling

### R-05: Mis-scoped query keys
- Mitigation: centralize keys in `src/utils/queryKeys.js` and forbid ad hoc inline key patterns
