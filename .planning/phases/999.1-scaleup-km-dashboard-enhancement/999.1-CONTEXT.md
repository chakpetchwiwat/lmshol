# Phase 999.1: Scaleup KM Dashboard Enhancement - Context

**Gathered:** 2026-04-17
**Status:** Ready for planning

<domain>
## Phase Boundary

This phase delivers the strategic analytical layer of the "KM + Performance" vision. It transforms raw quiz/point data into executive-level insights: Skill Gap Analysis, Department Benchmarking, Risk Identification, and Incentive ROI.

</domain>

<decisions>
## Implementation Decisions

### Skill Gap Analysis & Logic
- **D-01: Competency Mapping**: Use a Strict Hierarchy where each Course/Lesson is mapped to exactly one of the 6 core KM categories (Business, Soft Skills, Functional, Leadership, Compliance, Digital).
- **D-02: Gap Visualization**: Use a Radar Chart with a Static 100% Target baseline for all users, regardless of tier.
- **D-03: Proficiency Calculation**: Use "Mastery" logic—the proficiency score in a category is determined by the highest (best) quiz attempt within that category's courses.
- **D-04: Category Health Metrics**: In the context of the radar chart, provide drill-down or side-metrics for Pass, Non-Pass, Attended, Not Attended, and Completion counts to show the "Health" of that specific skill area.

### Department Performance Benchmarking
- **D-05: Ranking Metric**: The primary metric for the Department Leaderboard is **Completion Rate (%)** (assigned vs. finished).
- **D-06: Analytical Pacing**: Support periodic reporting (This Month, This Quarter, Custom Range) to allow managers to track growth over time rather than just lifetime totals.

### Risk Management
- **D-07: At-Risk Trigger**: Users are flagged in the "Risk Identification" widget based on **Deadlines**—specifically those with assigned mandatory courses that are overdue or approaching their deadline (e.g., < 3 days).

### Incentive ROI Visualization
- **D-08: ROI Metrics**: Implement two primary visualizations:
    1. A **Dual-Axis Trend Chart** showing "Total Points Distributed" vs "Total Completion Rate" to visualize the correlation between rewards and output.
    2. A **Redemption Table** showing the most popular rewards and their associated user activity levels.

</decisions>

<guardrails>
## ⚠️ Critical Platform Rules (DO NOT BREAK)

### Quiz Flow
- **G-01: Result Container is MANDATORY** — After a user submits a quiz, the page MUST scroll to and display a large, prominent result card (`QuizSection.jsx`) showing the score percentage. A toast notification alone is NOT acceptable and must NOT replace the result container.
- **G-02: Unified Reset Button** — Both passed and failed states show a single action button inside the result container. "ทบทวนอีกครั้ง" (passed) and "ทำควิซอีกครั้ง" (failed) both call `setQuizResult(null); setAnswers({})` to reset state cleanly. There is NO separate review-banner or show/hide summary toggle.
- **G-03: No showSummary State** — The `showSummary` local state has been removed. The result container is always visible when `quizResult` is set. Do NOT re-introduce a toggle mechanism.

### Lesson Navigation
- **G-04: No Full-Page Flash** — When navigating between lessons, `LessonPlayer.jsx` must NOT unmount and show a full-screen blank/white loader. The layout (sidebar, media area) must remain visible. Use a soft overlay spinner (`absolute inset-0 z-50`) on top of the existing content instead.
- **G-05: Soft Loading Pattern** — The full-screen loader (`flex min-h-screen`) is reserved ONLY for the initial load when `lesson` is null. All subsequent lesson transitions must use the overlay approach.

### Lesson Player UI
- **G-06: White Tag Labels** — The "เอกสารประกอบ" and "บทความแบบจัดรูปแบบ" pill labels in `LessonMedia.jsx` must use `text-white` (not `text-primary-light`). These labels render on a dark background and must maintain legibility.

</guardrails>
