# Phase 999.1: Scaleup KM Dashboard Enhancement - Research

## Standard Stack

- **Charts**: Recharts v3.8.0 (Use `RadarChart` for Skill Gap and `ComposedChart` for ROI).
- **ORM**: Prisma (using PostgreSQL/SQLite).
- **Backend**: Express + Admin Service logic.

## Architecture Patterns

### Prisma Schema Extension
- **Competency Mapping**: Introduce `CourseCategory` to establish a strict hierarchy.
    - `model CourseCategory { id String @id @default(cuid()); name String; courses Course[] }`
    - Add `categoryId String?` and `category CourseCategory? @relation(fields: [categoryId], references: [id])` to the `Course` model.
- **Risk Tracking**: Add `deadline DateTime?` and `assignedAt DateTime @default(now())` to the `UserCourse` or a dedicated `CourseAssignment` model if it exists.

### Aggregation Logic
- **Mastery Calculation**:
    - Step 1: `SELECT MAX(score) FROM QuizAttempt WHERE userId = X AND courseId = Y`
    - Step 2: Average these maximums across all `courseId`s belonging to the target `categoryId`.
- **Department Benchmarking**: Group `UserCourse` by `user.departmentId` and calculate `(SUM(status == 'COMPLETED') / COUNT(*)) * 100`.

### ROI trend (Biaxial)
- Use `ComposedChart` with `yAxisId="left"` for Completion Rate and `yAxisId="right"` for Points Distributed.
- Align data points using a temporal key (e.g., `YYYY-MM` or `YYYY-QQ`).

## Don't Hand-Roll

- **Visualizations**: NEVER build custom SVG paths for radar or trend lines. Use Recharts' primitive components (`Radar`, `Line`, `Bar`).
- **Data Filtering**: Do not filter large datasets (thousands of rows) in memory on the frontend; use Prisma's `groupBy` and aggregation functions at the database level.

## Common Pitfalls

- **Empty Categories**: Always default category scores to `0` instead of `null` or `undefined` to prevent Recharts from crashing or rendering empty radar domains.
- **Z-Index Clipping**: Radar charts in modals or tabs sometimes have tooltips clipped; use `clipPath={false}` or ensure overflow is visible in the container.
- **Timezone Variance**: Ensure deadlines are compared using UTC on the server to prevent "flickering" risk identification lists across localized clients.
